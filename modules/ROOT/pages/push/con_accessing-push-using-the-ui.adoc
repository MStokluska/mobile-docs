= Accessing {unifiedpush-service} using the User Interface

The Administrative User Interface (AUI):

* Displays all existing Push Applications.
* Displays the _Variants_ tab of a Push Application
** Details such as the number of sent messages and the amount of registered devices are display.
* Displays details about the 3rd party Push Network.
* Allows you to manage a registered device
* Allows you to import and export device metadata
* Send a Push Notification from (what?) code
* Access {unifiedpush-service} analytics
* Access {unifiedpush-service} activity

To access {unifiedpush-service} using the AUI, it must be running and you also require OpenShift credentials to log in.
