= Understanding {unifiedpush-service} Analytics

{unifiedpush-service} collects metadata and payload, and this is accessed using {unifiedpush-service} Analytics screen.

IMPORTANT: {unifiedpush-service} *automatically* deletes the collected metadata and payload after 30 days by an out-of-hours batch job.

The Analytics screen can been seen in the following image:

image::ups-admin-analytics-clean.png[{unifiedpush-service} Analytics screen]

.App Analytics
[options="header"]
|====
|Title|Explanation
|Push Messages|The total number of Push Notification Messages sent to {unifiedpush-service} for dispatching to registered devices.
|Notification Dispatched|The total number of Push Notifications dispatched via Push Networks to registered devices.
|Notifications Opened|Number of notifications that were delivered to registered device and opened by a user.
|Average Open Rate|The result of Push Notifications Opened / Push Notification Dispatched.
|====

.Performance
[options="header"]
|====
|Title|Explanation
|Push Messages|The number of targeted devices for each Push Notification.
|Notification Dispatched|The number of opened apps for each Push Notification.
|====

.Push Network - Displays the different Push Networks
[options="header"]
|====
|Platform|Instructions on how to activate _Push Notifications Opened_
|iOS|Refer to link:https://aerogear.org/docs/specs/aerogear-ios-push/Classes/AGPushAnalytics.html[AGPushAnalytics].
|Android|Refer to the link:https://aerogear.org/docs/specs/aerogear-android-push/[org.jboss.aerogear.android.unifiedpush.metrics] package.
|Cordova|Refer to link:https://aerogear.org/docs/guides/aerogear-cordova/AerogearCordovaPush/#_metrics[AerogearCordovaPush].
|====
